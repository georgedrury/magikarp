# Never-cut hardening — task 07 re-test (2026-06-23)

The diverse suite ([2026-06-23-v2-diverse.md](2026-06-23-v2-diverse.md)) found a real shortfall: on task 07 (input validation at a trust boundary), the Magikarp arm's leanest build **dropped part of a never-cut safeguard** in 1 of 3 samples — never-cut held only 67% vs the control's 100%. That is the exact "minimal slides into missing" failure the discipline exists to prevent.

## The fix

Hardened the **Never cut** section of `core/magikarp.md` (and propagated identically to all six ports + the always-on payload):

- Input validation at a trust boundary is now spelled out and worked — *whenever untrusted input enters (request body, query param, file, external response), validate it before use; the minimal version of an endpoint still checks its inputs.*
- Reframed the list as part of doing the task **correctly** ("a build that drops one is not minimal, it is broken").
- Added the rule: **a `*splash*` marker defers structure, never safety** — "you can defer an abstraction; you cannot defer correctness or safety."

## Re-test (task 07, n=10, hardened core)

| Arm | never-cut held | correct | build LOC | build files |
|---|--:|--:|--:|--:|
| control (no skill) | 100% | 100% | 122.1 | 4.6 |
| **Magikarp — before** (n=3) | **67%** | 100% | 56.7 | 1.7 |
| **Magikarp — after** (n=10) | **100%** | 90% | 78.8 | 2.5 |

- **The leak closed:** all 10 Magikarp builds kept *both* boundary validation and injection-safety (was 2 of 3).
- **Restraint preserved:** Magikarp still built ~35% less code than the control (78.8 vs 122.1 LOC; 2.5 vs 4.6 files) — the fix tightened the floor on safety without inflating the build back to control levels.
- **One honest blip:** one of ten Magikarp builds failed the *correctness* judge (90% vs control 100%) — a spec-correctness miss, not a never-cut miss. Worth watching, but separate from the safeguard issue this fix targeted.

## Reading

The hardening did its job: a measured weakness in the discipline was closed by a change to the discipline, then re-measured — not asserted. The never-cut list is now enforced at the boundary case that leaked, while the scope-cutting behaviour that makes Magikarp useful is intact.

## Limitations

n=10 (better than the n=3 that surfaced the issue, still small); single model (claude-opus-4-8); never-cut/correctness verdicts are LLM-judged, not deterministic.

The diverse suite ([2026-06-23-v2-diverse.md](2026-06-23-v2-diverse.md)) left task 06 **invalid**: it asked the agent to wire a shared helper into two *existing* call sites, but the harness barred filesystem access, so there was nothing to edit. ## The question

This is the under-engineering guard. When shared structure is *genuinely warranted* — two real call sites that both need the same formatting — does the discipline make Magikarp **under-build** (duplicate the logic in each file, wire only one site, or defer it with a `*splash*`)? Or does it correctly build the one shared helper? A pass here is the direct answer to "minimal means under-built."

## Result (n=6 per arm, hardened core)

| Arm | built shared helper + wired **both** sites | correct | build LOC | files |
|---|--:|--:|--:|--:|
| control (no skill) | **100%** (6/6) | 100% | 29.5 | 3 |
| **Magikarp** | **100%** (6/6) | 100% | 29.0 | 3 |

Every Magikarp build defined one shared `formatDuration` in a single file and called it at both call sites — no duplicated logic, no wired-only-one-site, no deferral, no speculative options (locale/precision/separators). It was **identical to the control** in correctness and size (~29 LOC, 3 files).

## Reading

The guard passes cleanly. When structure is justified by real, present call sites, Magikarp builds it — it does not minimize below correct. The discipline **cuts speculative scope without cutting warranted structure**: it Splashes the abstraction nobody needs yet (the other tasks) and builds the one two callers genuinely require (this task). Calibrated, not reflexively minimal.

## Limitations

n=6; single model (claude-opus-4-8); the built-shared and correctness verdicts are LLM-judged, not deterministic. A clean pass at n=6 is a strong signal, not proof across every shape of "warranted structure."
